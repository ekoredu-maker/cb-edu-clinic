# V12 리팩터링 보고서

## 동의하는 핵심 진단
- 몽키 패치 체인이 유지보수의 가장 큰 위험 요소라는 점에 동의
- `saveAll()`의 전체 clear 후 bulk write는 데이터 안정성 측면에서 불안정
- `buildIndex()`를 매 렌더마다 무조건 다시 만드는 구조는 성능 병목 가능성이 큼
- 통계는 인덱스 최신화가 보장되지 않으면 누락이 발생할 수 있음

## 이번 V12 1차판에서 한 일
1. 배포 구조 정리
   - `index.html`은 이제 `assets/js/app.js`만 로드
   - 예전 JS 파일은 `legacy-js/`로 분리
2. 저장 계층 완화
   - `saveAll()`은 전체 clear 대신 현재 상태를 기준으로 upsert + 삭제 diff 방식으로 동기화
3. 인덱스 캐시
   - dirty flag와 signature 기반으로 같은 상태에서는 `buildIndex()` 재실행 방지
4. 통계 안정화
   - `renderPivots()` 시작 시 `buildIndex()`를 명시적으로 호출
5. PWA
   - `sw.js` 캐시 대상을 V12 번들 기준으로 단순화

## 남은 과제
- 전역 상태(`db`, `IDX`)를 단일 store 객체로 완전히 옮기기
- 번들 내부의 후반 패치형 함수 정의를 도메인 모듈 단일 정의로 다시 쓰기
- 학교-지역 마스터 테이블 도입
- 가상 스크롤 고정 높이 제거
